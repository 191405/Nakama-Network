## 🎯 When to Edit Each File

### Common Edits

**To change colors:**
- Edit `tailwind.config.js` colors section

**To add new page:**
1. Create `src/pages/YourPage.jsx`
2. Add route in `src/App.jsx`
3. Add nav link in `src/components/Navbar.jsx`

**To modify Firebase functions:**
- Edit `src/utils/firebase.js`
- Add new functions as needed

**To add AI features:**
- Edit `src/utils/gemini.js`
- Create new prompt functions

**To change styling:**
- Edit `src/index.css` for global styles
- Use Tailwind classes in components

**To modify ranking system:**
- Edit `calculateRank()` in `firebase.js`
- Update thresholds and tier names

### Advanced Edits

**To add new database collection:**
1. Add Firestore functions in `firebase.js`
2. Update security rules in Firebase Console
3. Create UI components for CRUD operations

**To integrate new API:**
1. Add API key to `.env.example`
2. Create utility file in `utils/`
3. Import and use in components

**To add authentication provider:**
1. Enable in Firebase Console
2. Add button in `Login.jsx`
3. Add auth function in `firebase.js`

## 💡 Pro Tips

1. **Never commit .env** - It's in .gitignore for security
2. **Use ESLint** - Keep code clean and consistent
3. **Component reuse** - Extract repeated UI into components
4. **Error boundaries** - Add error handling to pages
5. **Loading states** - Always show feedback to users
6. **Mobile first** - Test on mobile devices regularly
7. **Git commits** - Commit often with clear messages
8. **Documentation** - Update docs when changing features

## 🆘 Common Issues

**"Module not found"**
- Check imports match file names exactly
- Ensure file extensions are correct (.jsx not .js for React)

**"Firebase is not defined"**
- Check .env file exists and has correct keys
- Verify imports in firebase.js

**"Build failed"**
- Run `npm install` again
- Delete node_modules and reinstall
- Check for syntax errors in JSX

**"Styles not applying"**
- Restart dev server
- Check Tailwind config
- Verify class names are correct
